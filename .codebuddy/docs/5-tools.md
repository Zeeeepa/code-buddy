# Tool System

The Tool System is a foundational component of the project, enabling the agent to interact with its environment, execute commands, and retrieve information. This system employs a sophisticated dual-registry architecture combined with RAG (Retrieval-Augmented Generation)-based selection, which is critical for maintaining efficiency and scalability as the number of available tools grows. This design ensures that the agent can dynamically access the most relevant tools for any given task, optimizing performance and reducing computational overhead.

## Tool Registry

The Tool Registry serves as the central catalog for all available tools within the system. It provides a structured mechanism for defining, organizing, and discovering tools, which is essential for managing the project's extensive tool ecosystem. Tools are primarily organized within the `src/tools/` and `src/tools/registry/` directories, ensuring a clear separation of concerns and ease of maintenance. The registry abstracts the underlying implementation details, allowing the agent to interact with tools through a consistent interface.

The tool ecosystem contains **117** tool modules organized in `src/tools/` and `src/tools/registry/`.
